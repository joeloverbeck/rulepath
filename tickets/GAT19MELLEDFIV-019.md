## Problem

Meldfall Ledger must be reachable from the browser through the Rust↔WASM bridge: catalog metadata (id, display name, seat range, default seats, bot levels, public-observer support, rules path), setup with seat-count validation, viewer-scoped view export, redacted legal action tree, Rust-safe previews, apply-action with effect groups, bot action, and viewer-scoped replay export/import. It also authors `HOW-TO-PLAY.md` (with the hidden-information section), generates the player-rules markdown + manifest, and registers `meldfall_ledger` in `check-player-rules`'s `HIDDEN_INFO_GAMES`.

## Assumption Reassessment (2026-06-25)

1. `crates/wasm-api/src/games/{blackglass,river,vow}.rs` are per-game wrappers; `constants.rs` holds `GAME_*` + `GAME_*_DISPLAY_NAME` consts, `catalog.rs`/`games.rs`/`lib.rs` dispatch via a `RegisteredGame` enum (confirmed during reassessment). `scripts/copy-player-rules.mjs` and `scripts/check-player-rules.mjs` both key off the `constants.rs` `GAME_*` catalog const (not `ci/games.json`), and `HIDDEN_INFO_GAMES` in `check-player-rules.mjs` lists `river_ledger`/`vow_tide`/`blackglass_pact`.
2. Spec §4.3 (WASM row), Appendix E.1 (Rust/WASM contract), and §4.2 (HOW-TO-PLAY.md row) define the surface; the game behavior + viewer-scoped exports exist from GAT19MELLEDFIV-004–014.
3. Cross-artifact: the wasm catalog const is the source of truth for `copy-player-rules`/`check-player-rules`/`check-catalog-docs`/`check-outcome-explanations`. Adding the const opens those validators' red windows; `check-player-rules` closes here (HOW-TO-PLAY + generated rules + HIDDEN_INFO all co-land), while `check-catalog-docs` (GAT19MELLEDFIV-021) and `check-outcome-explanations` (GAT19MELLEDFIV-020) close later.
4. FOUNDATIONS §2 behavior authority: the WASM bridge exposes Rust-owned legal actions/previews/effects/exports; TypeScript decides no legality. Per `apps/web/scripts/smoke-load-wasm.mjs`/`smoke-ui.mjs` (hardcoded per-game catalog assertions), those harnesses gain a `meldfall_ledger` assertion.
5. FOUNDATIONS §11 no-leak: the WASM view/action/preview/export must not expose internal stock order or other hands (the GAT19MELLEDFIV-013 firewall is the source); `WASM must not expose privileged debug state` (spec Appendix F.1). HOW-TO-PLAY's hidden-information section is required for `HIDDEN_INFO_GAMES`.
